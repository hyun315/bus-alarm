# 버스 도착 알리미 — 배포 가이드

## 구성
- `index.html` : 화면 (정류소/노선/알림간격 설정 + 실시간 도착정보 표시)
- `api/bus.js` : 서울시 버스 API를 대신 호출하는 서버 함수 (CORS 우회 + 인증키 보관)

## 배포 방법 (Vercel)

1. 이 폴더를 그대로 GitHub 저장소에 올리거나, Vercel CLI로 배포합니다.
   ```
   npm i -g vercel
   cd bus-alarm
   vercel
   ```
2. Vercel 프로젝트 대시보드 → **Settings → Environment Variables**에서 아래 값을 추가합니다.
   - `BUS_SERVICE_KEY` = 공공데이터포털에서 발급받은 **Decoding 인증키**
3. 재배포(Redeploy)하면 `https://프로젝트명.vercel.app` 주소로 접속 가능합니다.

## 앱 사용법

1. 배포된 주소로 접속
2. "프록시 서버 주소"에 위에서 배포한 주소를 입력 (같은 도메인이면 비워둬도 됨)
3. 정류소 고유번호(arsId), 버스 번호, 알림 간격(분) 입력 후 **저장하고 알림 시작**
4. 브라우저가 알림 권한을 물으면 **허용**을 눌러야 실제 알림이 옵니다
5. 설정한 간격마다 자동으로 도착 정보를 확인해 알림을 보내드립니다

## 참고

- 브라우저 알림(Notification API)은 **탭을 열어둔 상태**에서만 동작합니다. 완전히 앱을 백그라운드/종료 상태에서도 울리게 하려면 추후 PWA + Push 구독(Service Worker) 방식으로 확장이 필요해요. 지금 버전은 "탭을 켜두고 출근/이동 준비하며 확인" 용도에 맞춘 심플한 버전입니다.
- 정류소에 여러 방향(상행/하행)의 같은 번호 버스가 있는 경우, 첫 번째로 매칭된 노선 정보를 보여줍니다. 필요하면 방향 정보로 구분하는 기능을 추가할 수 있어요.
